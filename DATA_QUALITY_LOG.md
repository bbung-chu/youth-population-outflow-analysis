# 데이터 품질 이슈 처리 로그

청년 인구 순유출 분석(STEP 1) 전처리 과정에서 발견한 데이터 품질 이슈와 처리 과정을 기록한다.
사용 데이터: KOSIS 국내인구이동통계 (시군구/성/연령별 순이동, 2015~2025)

각 이슈는 **증상 → 원인 → 해결 → 검증** 순서로 기록한다.

## 이슈 1. 유령 행정구역 코드 (22개)

**증상**
전체 250개 행정구역 코드 중 22개가 분석 기간(2015~2025) 전체에서 값이 0 또는 결측이었다.

**원인**
마산시·진해시(2010년 창원시로 통합), 여천시·여천군(1998년 여수시로 통합), 용인군·이천군·안성군·파주군·포천군·화성군(1990년대 시 승격으로 명칭 변경) 등 2015년 이전에 이미 폐지되거나 명칭이 바뀐 옛 행정구역 코드가 KOSIS 마스터 데이터에 그대로 남아 있었다. "국외 및 기타" 같은 비국내 지역 코드도 포함되어 있었다.

**해결**
`parent`(상위 시/도) + `행정구역명` 조합을 기준으로 그룹화한 뒤, 전 기간 값이 0 또는 결측인 조합을 탐지하여 제거했다.

```python
ghost_regions = (
    df.groupby(["parent", "행정구역(시군구)별"])[year_cols]
    .apply(lambda g: (g.fillna(0) == 0).all().all())
)
ghost_keys = set(ghost_regions[ghost_regions].index)

df["_key"] = list(zip(df["parent"], df["행정구역(시군구)별"]))
df = df[~df["_key"].isin(ghost_keys)].drop(columns="_key").copy()
```

**검증**
제거 전 10,298행 → 제거 후 9,462행. 제거된 22개 지역 목록을 출력하여 전부 실제로 폐지·통합된 옛 행정구역임을 확인했다.

> 처음에는 지역명만으로 그룹화(`groupby("행정구역(시군구)별")`)했는데, 이 경우 그룹 결과 인덱스가 문자열이 되어 이후 필터링이 정상 작동했다. 이후 그룹 키를 `["parent", "행정구역명"]`으로 바꾸는 과정에서, 그룹 결과 인덱스가 튜플로 바뀌었는데 필터링 코드는 문자열 비교를 그대로 사용해 아무것도 제거되지 않는 문제가 있었다. 컬럼을 조합한 키(`_key`)를 명시적으로 만들어 비교하는 방식으로 해결했다.

## 이슈 2. 동명 행정구역 충돌 (서울 중구 vs 울산 중구 등)

**증상**
원본 데이터에 시/도를 나타내는 별도 컬럼이 없고, "중구", "동구", "남구", "서구", "북구" 같은 이름이 서울·부산·대구·인천·대전·울산 등 여러 광역시에서 반복해서 사용되고 있었다. 지역명만으로 집계하면 서로 다른 도시의 동명 구가 하나로 합산되는 문제가 있었다.

**원인**
원본 표가 "시/도 행 → 그 소속 구/군 행들"이 순서대로 나열된 계층 구조였는데, 이 순서 정보가 컬럼이 아니라 행 순서에만 담겨 있었다.

**해결**
시/도 이름 목록을 정의하고, 해당 이름이 나올 때마다 갱신되는 `parent` 컬럼을 forward-fill로 만들어 각 구/군 행에 소속 시/도를 부여했다. 이후 `parent + region` 조합을 지역의 고유 키로 사용했다.

```python
parents = ['서울특별시', '부산광역시', ..., '제주특별자치도']
df["parent"] = df["행정구역(시군구)별"].where(df["행정구역(시군구)별"].isin(parents)).ffill()
```

**검증**
`parent + 행정구역명` 조합 기준 고유 지역 수(231개)가 이름만 기준으로 셌을 때(211개)보다 많다는 것을 확인해, 실제로 동명 구가 여러 도시에 걸쳐 존재함을 검증했다. "중구"의 경우 서울특별시와 울산광역시 두 곳에 존재함을 확인했다.

## 이슈 3. 군위군 관할 변경 (2023년 7월 1일, 경상북도 → 대구광역시)

**증상**
"군위군"이라는 이름의 행이 두 개 존재했다. 하나는 2015~2022년만 값이 있고 2023년은 0, 나머지는 결측이었고, 다른 하나는 2023~2025년만 값이 있고 이전 연도는 결측이었다.

**원인**
군위군은 2023년 7월 1일 실제로 경상북도에서 대구광역시로 편입되었다. 경계 연도(2023)에는 이관 처리로 인해 기존 관할(경상북도) 쪽 값이 0으로 기록된 것으로 판단된다.

**해결**
군위군은 관할 자체가 바뀐 사례이므로, 하나의 지역으로 통합하지 않고 관할 시기별로 분리된 상태를 유지하기로 결정했다. 다만 관할 이관에 따른 플레이스홀더인 경상북도 군위군의 2023년 값은 결측으로 처리하여 실제 값과 구분했다.

```python
gunwi_gyeongbuk = (df["parent"] == "경상북도") & (df["행정구역(시군구)별"] == "군위군")
df.loc[gunwi_gyeongbuk, "2023 년"] = np.nan
```

이후 청년층 데이터 추출 단계에서 경상북도 군위군의 2023년 행을 명시적으로 한 번 더 제거해 중복 방지 안전장치를 두었다.

**검증**
최종 결과에서 "경상북도 군위군"은 2015~2022년(8개 연도), "대구광역시 군위군"은 2023~2025년(3개 연도)으로 정확히 분리되어 나타나는 것을 확인했다.

> 처음에는 `np.nan` 할당 시 컬럼명을 `"2023년"`(공백 없음)으로 잘못 입력해 실제로는 존재하지 않는 새 컬럼을 만드는 데 그쳤다. 다행히 이후 melt 이후 단계에서 `parent`/`region`/`year` 기준으로 한 번 더 걸러내는 명시적 필터가 있어 최종 결과에는 영향이 없었지만, 원래 의도한 위치에서 처리되도록 컬럼명을 `"2023 년"`(공백 포함)으로 수정했다.

## 이슈 4. 인천 남구 → 미추홀구 개명 (2018년 7월 1일)

**증상**
"남  구"(인천)의 값이 2015~2018년까지만 있고 2019년 이후는 0, "미추홀구"의 값은 2015~2017년은 0이고 2018년부터 값이 있었다. 개명 병합 코드를 작성했는데도 병합이 되지 않고 그대로 남아있었다.

**원인**
인천 남구는 2018년 7월 1일 미추홀구로 개칭되었다 — 행정구역 관할이 바뀐 것이 아니라 단순 개명이므로 군위군과 달리 하나의 연속된 지역으로 병합하는 것이 맞다. 병합 코드가 동작하지 않은 이유는 원본 데이터의 지역명이 `"남구"`가 아니라 `"남  구"`(공백 2칸)로 저장되어 있었기 때문이다. 서울·부산·대구·광주·대전 등 다른 광역시도 구 이름 중간에 공백이 들어가 있는 경우가 있어, 정확한 문자열을 `repr()`로 직접 확인하지 않으면 조건문이 조용히 아무것도 매칭하지 못한다.

**해결**
```python
mask_namgu = (df["parent"] == "인천광역시") & (df["행정구역(시군구)별"] == "남  구")
df.loc[mask_namgu, "행정구역(시군구)별"] = "미추홀구"

mask_michuhol = (df["parent"] == "인천광역시") & (df["행정구역(시군구)별"] == "미추홀구")
michuhol_merged = (
    df[mask_michuhol]
    .groupby(["parent", "행정구역(시군구)별", "성별", "연령별"], as_index=False)[year_cols]
    .sum(min_count=1)
)
df = pd.concat([df[~mask_michuhol], michuhol_merged], ignore_index=True)
```

**검증**
병합 후 2015~2017년이 실제 값으로 채워지고, 두 이름 모두 값이 존재했던 전환 연도(2018년)는 552 + (-278) = 274로 정상 합산되는 것을 확인했다.

## 배운 점

- KOSIS처럼 여러 기관·연도에 걸쳐 누적된 공공 통계는 행정구역 개편사(승격, 통합, 편입, 개명)를 데이터 정합성 문제로 직접 마주치게 된다. 전처리 전에 대상 기간 동안의 행정구역 변경 이력을 먼저 확인하는 것이 효율적이다.
- 지역명, 컬럼명처럼 겉보기에 단순한 문자열 비교는 원본 데이터의 공백·인코딩을 `repr()`로 직접 확인하지 않으면 조용히 실패할 수 있다. 조건문 작성 후에는 항상 "몇 건이 매칭되었는지" 카운트를 출력해 검증하는 습관이 필요하다.
- groupby 결과의 인덱스 타입(문자열 vs 튜플)이 그룹화 키 개수에 따라 달라진다는 점을 인지하지 못하면, 필터링이 아무 효과 없이 통과되는 버그가 발생할 수 있다.
